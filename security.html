Windows AD Server.

Updates.

De status van de Windows Server is up-to-date.

Op de server is er ingesteld dat er automatisch wordt gezocht naar nieuwe updates en als deze er zijn worden deze automatisch geïnstalleerd.

Firewall.

Op de Windows server wordt de Windows Firewall gebruikt en deze staat aan voor zowel Domain networks, Private networks en Guest or public networks. Op de server is alleen de optie Domain network connected. Dit is connected met het domein Test.local, hierbij worden alle connecties naar apps die niet op de toegestane lijst staan geblocked.

Backups.

Op de Windows Server worden geen back-ups gemaakt. Ook zijn er geen bestaande back-ups op de server gevonden.

  Windows Member Server.

De status van de Windows Member Server is niet up-to-date.

Op de server is niet ingesteld dat er automatisch wordt gezocht naar nieuwe updates en als deze er zijn worden deze dan ook niet geïnstalleerd. Dit is ook de reden waarom de server niet up-to-date is.

  Conclusie.

Na de analyse van beide systemen ben ik tot de volgende conclusies gekomen:

• De systemen zijn op een systeem na goed ingesteld voor updates.

• Op beide systemen wordt er gebruik gemaakt van de firewall.

• Op beide systemen wordt er geen gebruik gemaakt van een specifiek anti-malware programma. Wel wordt er op beide systemen gebruik gemaakt van Windows Defender, alleen wordt daar bij de servers geen gebruik gemaakt van de Windows Smartscreen en de Network Access Protection.

• Op beide systemen wordt er geen gebruik gemaakt van back-ups en er zijn ook geen back-ups gemaakt in het verleden.

Zoals te zien is aan de bovenstaande punten, zijn de systemen niet security proof. Om dit toch voor elkaar te krijgen, heb ik de volgende aanbevelingen:

• Stel op elke systeem automatisch updaten in. Er hoeft dan niet meer zelf gekeken te worden of er updates zijn en of deze geïnstalleerd zijn. Ook is het dan niet erg als je zelf vergeet om te controleren op updates, want dit wordt al vanzelf gedaan.

• Om te zorgen dat alle apps ook up-to-date zijn, is het handig om het programma “Patch My PC” te gebruiken. Dit programma laat in een keer zien welke apps up-to-date zijn en welke niet en zal dan vervolgens ook gelijk de niet up-to-date apps installeren.

• Stel op de Windows Servers ook de opties Windows Smartscreen en de Network Access Protection in bij Windows Defender.

• Kijk ook naar een goede anti-malware programma voor het scannen en verwijderen van malware. Kaspersky is hier een goed programma voor.

• Zorg er ook voor dat er op beide systemen back-ups gedraaid wordt. Dit voorkomt een hoop problemen in de toekomst. Hanteer hierbij de 3-2-1-regel: zorg voor drie kopieën van de data, op twee verschillende apparaten, waarvan een buitenshuis. Zorg er ook voor dat de back-ups automatisch worden gedaan bij voorkeur s ’nachts.

Concluderend, nee alle systemen zijn niet goed beveiligd. Maar als de genoemde punten worden aangepast en er gebruik wordt gemaakt van de bovenstaande aanbevelingen, zullen de systemen een stuk veiliger worden.
